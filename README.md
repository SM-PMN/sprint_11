# Спринт 11

Практическая работа по автоматизации развёртывания инфраструктуры в Yandex Cloud с помощью Terraform и Ansible.

## Цель работы

В рамках проекта необходимо:

1. Развернуть с помощью Terraform три виртуальные машины с доступом по SSH через ранее созданный ключ.
2. Настроить подключение Ansible к этим машинам по тому же SSH-ключу.
3. Реализовать три Ansible-роли:
   - базовая настройка ВМ;
   - настройка проксирующего Nginx;
   - настройка backend Nginx.

## Что реализовано

Проект автоматизирует полный цикл подготовки инфраструктуры:

- Terraform создаёт сеть, подсеть и три виртуальные машины в Yandex Cloud.
- На все ВМ передаётся публичный SSH-ключ через `metadata`.
- Ansible использует inventory с группами `proxy` и `backend`.
- Роль `common` меняет hostname на имя хоста из inventory и устанавливает базовые пакеты: `net-tools`, `dnsutils`, `wget`, `curl`, `mc`, `rsync`.
- Роль `nginx_backend` устанавливает Nginx на backend-серверы и разворачивает страницу с текстом `Hello from <имя машины>!`.
- Роль `nginx_proxy` устанавливает и настраивает Nginx на proxy-сервере, который слушает порт `3000` и проксирует запросы на backend-серверы по порту `80`.

## Структура проекта

`Terraform/` - описание инфраструктуры в Yandex Cloud:

- `providers.tf` - провайдер и требования к версии Terraform;
- `variables.tf` - переменные проекта;
- `main.tf` - сеть, подсеть, диски и виртуальные машины;
- `outputs.tf` - внутренние и внешние IP-адреса созданных ВМ;
- `tfenv.sh` - экспорт переменных окружения для работы с Yandex Cloud CLI.

`Ansible/` - конфигурация серверов после создания инфраструктуры:

- `inventory/hosts.yml` - inventory с группами `proxy` и `backend`;
- `playbook.yml` - основной playbook;
- `roles/common` - базовая настройка всех ВМ;
- `roles/nginx_backend` - настройка backend Nginx;
- `roles/nginx_proxy` - настройка reverse proxy Nginx.

## Логика работы

После применения Terraform создаются три ВМ:

- одна машина выполняет роль `proxy`;
- две машины выполняют роль `backend`.

Затем Ansible применяет роли:

1. Ко всем хостам применяется `common`.
2. К группе `proxy` применяется `nginx_proxy`.
3. К группе `backend` применяется `nginx_backend`.

В результате входящий запрос на proxy-сервер по порту `3000` перенаправляется на один из backend-серверов, который отдаёт HTML-страницу с именем текущего хоста.

## Как запускать

### 1. Подготовка Terraform

При необходимости экспортировать переменные окружения для Yandex Cloud:

```bash
cd Terraform
source tfenv.sh
```

Инициализация и применение конфигурации:

```bash
terraform init
terraform plan
terraform apply
```

### 2. Настройка серверов через Ansible

После создания ВМ и проверки актуальности IP-адресов в `Ansible/inventory/hosts.yml`:

```bash
cd Ansible
ansible-playbook playbook.yml
```

## Ожидаемый результат

- Все виртуальные машины доступны по SSH.
- На всех серверах изменены hostname и установлены базовые утилиты.
- На backend-серверах работает Nginx и отображается страница `Hello from <имя машины>!`.
- На proxy-сервере работает Nginx reverse proxy на порту `3000`.
- При обращении к proxy-серверу пользователь получает ответ от одного из backend-серверов.
